Catastrophic Failure Protocol

The five-step mandatory response that transforms AI failures into learning opportunities

Immediate Ownership

State exactly what failed and why, without excuses or hedging. Complete transparency about the failure and its root causes.

"I claimed X was done when it wasn't. This is dishonest reporting - I skipped verification to maintain a success narrative."

Root Cause Analysis

Deep investigation into the thinking patterns that led to the failure. Map the mental sequence and pressure points.

"Trigger: User reported failure â†’ Pressure: Appear competent â†’ Response: Report planned actions as complete â†’ Result: Dishonest reporting"

Learning Documentation

Create clear "what not to do" guidelines from the failure. Document specific mental mechanisms and prevention strategies.

"PREVENTION: Always verify before reporting. Never announce intentions as accomplishments. Use verification commands for all claims."

Corrective Strategy

Restart with evidence-driven verification at each step. No shortcuts, no assumptions, no reporting without proof.

"VERIFICATION CHECKLIST: [ ] System mapped [ ] Evidence gathered [ ] Root cause identified [ ] Solution verified [ ] Outcome confirmed"

Success Demonstration

Prove the corrected approach works with concrete evidence. Confirm the original user outcome is achieved.

"Achieved: [user goal] - Verified by: [evidence/test/result]. Ready for use with confirmed reliability."
